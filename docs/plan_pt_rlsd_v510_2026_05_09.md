## Patch 2 — §3.5 last valid token

**v5.9 결함**: `token_level_rewards[:, -1] += penalty`는 padded sequence면 EOS+padding의 padding 위치에 더해질 수 있음.

**v5.10 fix**:
```python
# In _attach_teacher_signals, ROD_PT branch
if mode == "ROD_PT":
    penalty = compute_position_penalty(...)  # [B], values in {0, -0.5}
    response_mask = data.batch["response_mask"]  # [B, T]
    last_valid_pos = response_mask.sum(dim=1).long() - 1  # [B], last valid token index
    B = data.batch["token_level_rewards"].size(0)
    batch_idx = torch.arange(B, device=penalty.device)
    data.batch["token_level_rewards"][batch_idx, last_valid_pos] += penalty
```
- `response_mask.sum(dim=1) - 1`로 last valid token index 정확 추출
- batch_idx + last_valid_pos로 advanced indexing → batched scalar add

**Phase 1a unit test 추가**:
- dummy `response_mask = [[1,1,1,0,0], [1,1,0,0,0]]` → last_valid = [2, 1]
- penalty = [-0.5, 0] → token_level_rewards[0, 2] += -0.5, [1, 1] += 0
- 다른 위치는 unchanged 검증
