### 3.3 META_START append trick — R5 builder 직접 호출 (round 5 R3 fix)

**v5.15 결함**: 별도 `_build_position_teacher_batch` 작성이 R5 mirror 아님.

**v5.16 fix — R5 `_build_teacher_logprob_batch` 직접 호출 + answer suffix**:

```python
# In _attach_teacher_signals, ROD_PT branch
if mode == "ROD_PT":
    META_START_TOKEN = tokenizer.decode([meta_start_id], skip_special_tokens=False)

    # Step 1: For each rollout, find first META_START position p in response.
    # Build "truncated rollout" where rollout response is replaced by completion[:p].
    truncated_responses = []
    rollout_indices = []
    for b in range(B):
        valid = (response_tensor[b] == meta_start_id) & response_mask[b].bool()
        nz = valid.nonzero(as_tuple=True)[0]
        if nz.numel() == 0:
            continue
        p = int(nz[0].item())
        # Trim response to [:p] then append META_START token
        trimmed_ids = response_tensor[b, :p][response_mask[b, :p].bool()]
        # Re-pad to original response_length with PAD, but record actual position p
        truncated = torch.zeros_like(response_tensor[b])
        truncated[:p] = response_tensor[b, :p]
        truncated[p] = meta_start_id  # ensure META at position p (already there from rollout)
        truncated_responses.append(truncated)
        rollout_indices.append(b)

    if not truncated_responses:
        # No META emit anywhere — w_position = 1.0 default (no amplify)
        data.batch["sdc_position_log_prob_meta"] = torch.zeros(B, device=target_device)
    else:
        # Step 2: Use R5 builder with answer_texts = "gold" (no extra suffix)
        # responses[b][:p] already ends with META_START token (it's IN the response at p)
        # → ref_log_prob[b, p] = log_prob(META_START | prompt + answer + completion[:p])
        N = len(rollout_indices)
        truncated_response_tensor = torch.stack(truncated_responses, dim=0)  # [N, T_response]

        # Build mask: 1 up to and including position p, 0 after
        truncated_mask = torch.zeros_like(response_mask[rollout_indices])
        for i, b_orig in enumerate(rollout_indices):
            valid = (response_tensor[b_orig] == meta_start_id) & response_mask[b_orig].bool()
            nz = valid.nonzero(as_tuple=True)[0]
            p = int(nz[0].item())
            truncated_mask[i, :p+1] = 1.0  # include META at p

        position_batch = _build_teacher_logprob_batch(  # R5 builder 직접 호출
            tokenizer=tokenizer,
            prompt_texts=[prompt_texts[b] for b in rollout_indices],
            answer_texts=[gold_answers[b] for b in rollout_indices],  # gold만 (META는 response에서)
            responses=truncated_response_tensor,                       # [N, T_response]
            response_mask=truncated_mask,                              # [N, T_response] mask up to p
            v0_prefixes=None,
            forced_meta=False,                                         # ROD-PT는 자연 emit
        )
        position_batch.meta_info["temperature"] = rollout_temp
        pos_out = trainer._compute_ref_log_prob(position_batch)
        # pos_out.batch["ref_log_prob"] shape [N, T_response]
        # Extract log_prob at position p for each rollout
        ref_log_probs = pos_out.batch["ref_log_prob"]  # [N, T_response]
        log_prob_meta_partial = torch.zeros(N, device=target_device)
        for i, b_orig in enumerate(rollout_indices):
            valid = (response_tensor[b_orig] == meta_start_id) & response_mask[b_orig].bool()
            nz = valid.nonzero(as_tuple=True)[0]
            p = int(nz[0].item())
            # ref_log_prob[i, p-1] = log_prob of token at position p (next-token prediction shift)
            # Actually depends on R5 convention — verify via R7 prob measurement (Phase 1c)
            # CONSERVATIVE: use ref_log_prob[i, p] (covered by truncated_mask)
            log_prob_meta_partial[i] = ref_log_probs[i, p]

        # Map back to full batch
        full_log_prob_meta = torch.zeros(B, device=target_device)
        for i, b_orig in enumerate(rollout_indices):
            full_log_prob_meta[b_orig] = log_prob_meta_partial[i]
        data.batch["sdc_position_log_prob_meta"] = full_log_prob_meta
```

**핵심 변경 (round 5 R3 fix)**:
- 별도 builder 폐기. R5 `_build_teacher_logprob_batch` 직접 호출.
- response = rollout response (truncated mask로 p+1까지만 valid). META_START는 이미 response[p]에 있음.
- ref_log_prob에서 position p의 값 추출 = log_prob(META | prompt + gold + response[:p]).

```python
# Inside compute_sdc_gdpo_advantage, ROD_PT mode
log_prob_meta = batch["sdc_position_log_prob_meta"].to(device)  # [B]
clip_eps = float(config.get("sdc_clip_eps_w", 0.2))

# RLSD attribution factor (R5와 동일)
attr_log = torch.clamp(teacher_pos - student_logp, -clamp, clamp)
w_attr = torch.clamp(torch.exp(sign * attr_log), 1.0 - clip_eps, 1.0 + clip_eps)

# Position factor (NEW)
sign_per_seq = sign  # [B, 1] — already shaped from R5 line 264
w_position = torch.clamp(
    torch.exp(sign_per_seq * log_prob_meta.unsqueeze(1)), 1.0 - clip_eps, 1.0 + clip_eps
)  # [B, 1]

# Combined factor — PRODUCT (RLSD invariant 보존)
w_combined = w_attr * w_position  # broadcast [B, T] × [B, 1] → [B, T]

# R5 verl_sdc_utils.py:322-329 패턴 정확 mirror
lam = self._current_lambda()
factor = meta_mask * ((1.0 - lam) + lam * w_combined) + (1.0 - meta_mask)
advantages = seq_adv * factor * response_mask
advantages = verl_F.masked_whiten(advantages, response_mask) * response_mask
return advantages, advantages
```

**RLSD invariant 보존 검증**:
- `sign(seq_adv) > 0` (좋은 rollout) + `w_combined ∈ [0.64, 1.44]` (≈[clip²]) → factor ≈ [0.64, 1.44] in meta region
- `seq_adv * factor`: advantage **부호 유지**. magnitude만 modulate.
- masked_whiten 후 정규화 → mean=0, std=1. 그러나 group 내 sign distribution은 보존.

## 9. v5.16 self-critic + round 6 trigger

1. **§3.3 ref_log_prob[i, p] vs ref_log_prob[i, p-1]**: R5 `_compute_ref_log_prob` convention. response의 first valid token이 prompt의 다음 token일 가능성 → ref_log_prob[i, 0]이 response[0]의 log_prob인지 prompt 마지막 token의 log_prob인지 확인 필요. Phase 1c 측정 시 명확화.
