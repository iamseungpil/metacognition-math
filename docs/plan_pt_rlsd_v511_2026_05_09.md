## Patch 1 fix — FSDP unwrap

**v5.10 결함**: `actor.actor_module(input_ids).logits` 직접 호출이 FSDP wrap 후 동작 보장 못 함.

**v5.11 fix**:
veRL 0.7.1의 actor는 `FSDPActor` (또는 `MegatronActor`). FSDP wrap된 module의 forward는 다음 셋업에서만 안전:
- `with torch.no_grad():` (gradient 안 필요한 frozen forward)
- ZeRO-3는 자동 all-gather 처리 (FSDP는 default fully sharded)

코드 (verl_sdc.py에 추가, line ~340 근처 `_build_teacher_logprob_batch` 다음에):
```python
def _compute_position_top_k_hits(
    trainer, input_ids, attention_mask, meta_start_id, top_k=16
):
    """Forward through reference policy (frozen teacher), return [B] bool: META_START in top-K of last valid position."""
    # Use the same _ref_policy_wg that R5 uses for compute_ref_log_prob.
    # That call already handles FSDP gather internally; we just need full logits.
    # Workaround: compute logprobs first (existing path), then re-extract logits via direct forward
    # under no_grad. veRL's FSDPModule is a torch.nn.Module subclass — direct forward works.
    ref_module = trainer._ref_policy_wg.actor_module  # FSDP-wrapped frozen ref policy

    with torch.no_grad():
        # FSDP wrapped forward — automatic all-gather happens internally
        outputs = ref_module(
            input_ids=input_ids.to(ref_module.device),
            attention_mask=attention_mask.to(ref_module.device),
        )
        logits = outputs.logits  # [B, T, V]

    # Extract last valid position per batch
    last_valid_pos = attention_mask.sum(dim=1).long() - 1  # [B]
    B = input_ids.size(0)
    batch_idx = torch.arange(B, device=logits.device)
    last_logits = logits[batch_idx, last_valid_pos]  # [B, V]

    # Top-K check
    top_k_idx = last_logits.topk(top_k, dim=-1).indices  # [B, K]
    in_top_k = (top_k_idx == meta_start_id).any(dim=-1)  # [B] bool
    return in_top_k
```

**검증 (Phase 1a unit test)**:
- mock `_ref_policy_wg.actor_module` (CPU dummy module returning fixed logits)
- 3 case: meta_start_id in top-K → True; not in top-K → False; padded sequence (last_valid_pos < T-1) → 정확한 위치에서 logits 추출

**검증 (Phase 1b GPU smoke)**:
- 실 veRL stack에서 FSDP module forward가 NaN/Inf 없이 logits 반환 확인

## Patch 2 fix — device 정합

**v5.10 결함**: `torch.arange(B, device=penalty.device)` — penalty가 어느 device일지 불명.

**v5.11 fix**:
모든 indexing은 token_level_rewards의 device 기준으로 통일:
```python
if mode == "ROD_PT":
    penalty_per_rollout = compute_position_penalty(...)  # CPU or CUDA
    target_device = data.batch["token_level_rewards"].device

    response_mask = data.batch["response_mask"].to(target_device)
    last_valid_pos = (response_mask.sum(dim=1).long() - 1).to(target_device)
    B = data.batch["token_level_rewards"].size(0)
    batch_idx = torch.arange(B, device=target_device)

    penalty_on_target = penalty_per_rollout.to(target_device)
    data.batch["token_level_rewards"][batch_idx, last_valid_pos] += penalty_on_target
```

**검증 (Phase 1a unit test)**:
- penalty CPU + target_device CPU → no error
- penalty CUDA + target_device CUDA → no error
- penalty CPU + target_device CUDA → automatic .to(cuda) 처리 검증
