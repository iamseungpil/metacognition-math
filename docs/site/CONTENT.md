# Useful Metacognition for Math Reasoning — content source (authoritative; render this, do not invent numbers)

## TITLE / HERO
- Title: **Useful Metacognition for Math Reasoning**
- Subtitle: *Teaching an LLM to think about its thinking — and stopping it when that doesn't help.*
- One-line: We reward a model's `<|meta|>` reasoning (verify / redirect / state confidence) ONLY when it causally raises accuracy. The hard part: meta everywhere HURTS.

## INTENT (overall)
North-star: meta-cognition → accuracy, **only when useful**. Calibration (confidence matching real accuracy) is a partial sub-goal. Core obstacle: a model that ALWAYS does meta breaks easy problems (decoration) and derails hard ones.

The "sweet spot" (one picture): meta HURTS on easy (decoration), HELPS on medium (catches slips), HURTS on hard/AIME (derails). Reward shaping must make meta selective + productive.

## THE 7 RECURRING MECHANISMS (the conceptual backbone — color these, every method maps to ≥1)
1. ✅ **Selectivity** — emit meta only when needed → helps. (CF)
2. ❌ **Always-on** — no necessity check → meta fires everywhere → breaks already-solved problems. (PMI, gm)
3. ❌ **Form ≠ Behavior** — reward the SHAPE of meta (verify block) not real cognition → decoration with no causal substance. (cf_group)
4. ❌ **Correctness-coupling** — tie meta reward to the correctness sign → model makes meta over-confident → suppresses epistemic hedging (wait/hmm) → worse reasoning. (gm multiplicative; SDPO paper)
5. ❌ **Self-verification ceiling** — meta is generated by the SAME model that erred, drawn from the SAME distribution → it inherits the blind spot → confirms the error instead of catching it. (root cause of all)
6. ❌ **Easy data** — SFT on easy/decorative meta degrades the base's hard-math capability + gives no save/derail signal. (rv data)
7. ⚙️ **Inert-centering** — a group-CONSTANT reward, centered within its own member group (GRPO), averages to its own value → centers to 0 → signal vanishes silently. (asym_cf gate bug, now fixed by whole-group centering)

## METHOD DEEP-DIVE CARDS (formula + intuition + WHY — the substance)
For each: a KaTeX formula box, a one-line intuition, a colored "Why" callout (color = its mechanism).

- **PMI** — formula: R = logp(continuation | prefix+meta) − logp(continuation | prefix), mean_min over the byte-identical C-span, then − placebo (contentless meta). It scores the model's OWN continuation tokens (e.g. "So the answer is 16"), NOT a teacher-forced gold answer. Why (Always-on, red): a confident meta raises the whole continuation everywhere, so it fires even for decorative meta; it never checks whether meta was NEEDED, so emission drifts to ~1.0 and breaks problems the model already solves.
- **CF** — Why (Selectivity, blue): the counterfactual directly measures necessity. Already-solved ⇒ acc_with = acc_without ⇒ zero reward for emitting ⇒ over-penalty suppresses waste ⇒ meta fires only on the ~20% where it flips wrong→right. Small (+0.04) because the meta content is still correlated with the model's own errors (mechanism 5 — the ceiling).
- **cf_group** — Why (Form≠Behavior, orange): we rewarded the verify-block shape; the group counterfactual accurately detected it added no causal value and drove emission to zero. A "correct" negative — fake usefulness was filtered.
- **gm add vs mult** — Why (Correctness-coupling, purple): multiplying meta reward by the correctness sign punishes good meta on wrong rollouts → the model learns to drop hedging and assert confidently → epistemic verbalization collapses → reasoning degrades (SDPO). Additive keeps the meta signal independent → epistemic preserved.
- **asym_cf** — Why (Inert-centering ⚙️ → fixed): the gate value is group-CONSTANT; centered within its own member mask it averaged to itself → centered to 0 → the suppression signal silently vanished (rmeta_neg_rate=0, emit drifted to 0.92). Fix: center over the WHOLE group (non-emitting rows carry 0, breaking the symmetry) → the negative survives → emit stops drifting up. Confirmed live.
- **PMI-shift** — Why (it should break Form≠Behavior at the content level): it measures the meta's CAUSAL belief update (before vs after the meta content). Re-derivation / decoration produces no shift; a genuine catch produces a decoy→gold reversal. Rewarding only the reversal targets real cognition, not shape.

## CONTRASTIVE EVOLUTION (the through-line — horizontal timeline, the highlight)
Every method isolates the meta's effect by CONTRASTING two things; the contrast got sharper and more causal over time:
- **PMI**: gold vs (no-meta) baseline — "does meta correlate with good?"  [correlation]
- **CF**: meta-ON vs meta-OFF — "does meta change the OUTCOME?"  [outcome contrast]
- **gm**: gold vs decoy × meta vs placebo — "which answer does the meta favor?"  [double / answer contrast]
- **PMI-shift**: gold vs decoy × BEFORE vs AFTER meta — "did the meta CONTENT move belief toward the right answer?"  [temporal double contrast]
Insight: the measurement evolved correlation → outcome → causal content-level belief-change. Same tool (contrast), progressively isolating causation.

## CORRECTED MECHANISM FACTS (verified against code — PMI was previously mis-stated; do NOT reintroduce the error)
- **PMI** (the reward method) scores the MODEL'S OWN CONTINUATION (the tokens it generates AFTER the meta block), NOT a teacher-forced gold answer. with-arm = prefix + meta + continuation; without-arm = prefix + continuation. Per continuation token: delta = logP_with − logP_without over the byte-identical shared C-span; aggregate with mean_min (RLT: mean + α·min on clipped deltas); then SUBTRACT the placebo (contentless meta). Intuition: "does the meta make MY OWN following answer more likely?" WEAKNESS: a confident meta raises the whole continuation everywhere → fires even for decorative meta → always-on.
- **gm** TEACHER-FORCES gold AND decoy answers. gm = [logp(gold|meta) − logp(gold|placebo)] − [logp(decoy|meta) − logp(decoy|placebo)]. A difference-in-differences: (how much meta lifts gold) − (how much meta lifts decoy). Positive = meta favors gold OVER a plausible wrong answer = discrimination. Decorative meta lifts gold and decoy equally → gm ≈ 0.
- **PMI-shift** measures the gold−decoy belief gap at TWO POSITIONS of the SAME rollout. PMI_open = logp(gold) − logp(decoy) at meta-OPEN (body just BEFORE `<|meta|>`). PMI_close = same gap at meta-CLOSE (through `<|/meta|>`). shift = PMI_close − PMI_open. decoy→gold reversal = big reward; gold→decoy = big penalty (asymmetric). Decorative meta → gap doesn't move → shift ≈ 0.
- ★**NAME COLLISION**: "PMI" means TWO different quantities — (1) in the PMI METHOD it is the with-meta vs without-meta logp of the model's OWN continuation; (2) in PMI-SHIFT, "PMI_open/close" is the gold−decoy gap. Different things sharing a name.
- The three differ on TWO axes: **TARGET** (PMI = own continuation; gm & PMI-shift = gold vs decoy) × **CONTRAST AXIS** (PMI = meta on/off; gm = meta vs placebo [condition]; PMI-shift = before vs after meta [time]).

## SHARED WORKED EXAMPLE (illustrative — directionally correct, distill.pub style)
Problem: **7 − 3·(2 − 5)**. Body with a SIGN SLIP: "2 − 5 = −3, 3·(−3) = −9, 7 − 9 = −2" → leans toward the DECOY answer −2. Meta block: "wait — I'm SUBTRACTING 3·(2−5) = −9, so it's 7 − (−9) = 7 + 9 = 16, not 7 − 9. Sign error." Continuation after meta: "So the answer is 16." gold = 16, decoy = −2. Before meta logp(−2) > logp(16); after meta logp(16) > logp(−2). The SAME example flows through the three reward calculators (PMI / gm / PMI-shift).
- PRODUCTIVE meta (above): PMI positive, gm positive, shift positive (decoy→gold reversal).
- DECORATIVE meta ("confidence 0.78, looks right, will verify"): PMI stays POSITIVE (own answer still flows) but gm ≈ 0 and shift ≈ 0 (gold & decoy move together / gap doesn't move). What each reward CATCHES vs MISSES is the whole point.

## RESULTS FOR VISUALS
- ★Per-difficulty (base vs e21r-meta, matched data, ROBUST where noted) — this is the honest headline, NOT a single overall number:
  - gsm8k (easy): **tie** — base 0.93 / meta 0.93 (integer answers, grader-independent → robust).
  - aime (hard): **meta collapses** — base **0.37** / meta **0.13** (holds across EVERY grader → robust; the one clear degradation).
  - math500 (medium): **within grader noise** — base 0.63–0.72 / meta 0.64–0.76 (LaTeX parsing swings it ±10pp; cannot cleanly separate).
- ★The overall base-vs-meta gap is NOT robust — its SIGN flips by grader: old-grader meta +4.7pp / completion-math_verify base +5.0pp / union −0.7pp (tie). So any "meta is N pp below base" claim is a grading artifact. Honest statement: meta ≈ base on easy/medium, collapses on hard.
- Held-out Δ ranking (grader-ROBUST because it is a PAIRED with-vs-without diff under one grader): CF +0.040 (only significant +) > PMI +0.010 > gm-add −0.003 > gm-mult −0.029. (chips) — this is the trustworthy reward-quality signal, unlike the fragile absolute 1030 numbers.
- Grader caveat (callout): the math500 checker is unreliable on LaTeX answers in BOTH directions (e.g. it scored `\frac{14}{3}` and `(3,π/2)` wrong despite correct completions; it also marked malformed-brace correct answers wrong). On base, math500 ranges 0.63 (old) → 0.72 (completion-math_verify). The earlier "63→89 / base 90" figures were over-corrections. Never publish a single absolute number without the grader caveat.

## HONEST STATUS / NEXT
- ✓ Confirmed: conditional reward (gate + PMI-shift) REDUCES the harm meta does on hard problems (stops derail). PMI-shift gives bidirectional belief signal; the gate (fixed) blocks always-on drift.
- ✗ Open: beating base GRPO in ABSOLUTE accuracy, esp. AIME. That is a CAPABILITY gap (the meta SFT degraded the base) + the self-verification ceiling — reward shaping cannot create a capability the model lacks.
- → Next levers (beyond reward shaping): (a) preserve base capability (train meta WITHOUT easy-only data / from a strong base), (b) inject INDEPENDENCE (external tools, code-check, multi-sample consistency) to break the self-verification ceiling.
