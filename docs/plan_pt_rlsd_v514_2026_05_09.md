### 3.4 Position signal computation (정정)

```python
# In _attach_teacher_signals, ROD_PT branch
if mode == "ROD_PT":
    position_batch = _build_position_teacher_batch(
        tokenizer=tokenizer,
        prompt_texts=prompt_texts,
        gold_answers=gold_answers,
        responses=response_tensor,
        response_mask=response_mask,
        meta_start_id=tokenizer.convert_tokens_to_ids("<|meta|>"),
        rollout_temp=rollout_temp,
    )
    if position_batch is None:
        # No rollout has META emit — assign zero signal (driver fallback)
        data.non_tensor_batch["position_signal"] = np.zeros(B, dtype=np.float32)
    else:
        # Use existing R5 path — _compute_ref_log_prob reads input ending in META_START token
        # and returns log_prob of that META_START as the last token's log_prob.
        position_batch.meta_info["temperature"] = rollout_temp
        pos_out = trainer._compute_ref_log_prob(position_batch)
        # pos_out.batch["ref_log_prob"]: [N, T_response_only] — last position is META_START log_prob
        ref_log_probs = pos_out.batch["ref_log_prob"]
        prob_meta = torch.exp(ref_log_probs[:, -1])  # [N], prob_meta at last position
        rollout_indices = position_batch.non_tensor_batch["rollout_indices"]

        # Continuous signal — multiplicative pattern (RLSD invariant 보존)
        # signal = exp(sign(A) × log_prob_meta) clipped → multiplicative on meta region
        # 그러나 ROD-PT의 position penalty는 advantage 자체가 아니라 reward dim에 더하는 것
        # round 3 R5 권고: additive but small enough to not flip sign
        base = float(config.get("pt_position_penalty_base", 0.5))
        position_signal_raw = -base * (1.0 - prob_meta).numpy().astype(np.float32)

        # Map back to full batch
        full_signal = np.zeros(B, dtype=np.float32)
        for i, b_orig in enumerate(rollout_indices):
            full_signal[b_orig] = position_signal_raw[i]
        data.non_tensor_batch["position_signal"] = full_signal
```

**RLSD invariant 처리** (round 3 R5 결함):
- `position_signal ∈ [-0.5, 0]` (base=0.5 권고). correctness ∈ {0, 1} + meta_penalty ∈ {-0.2, 0} 합 = max 1.0. position_signal -0.5는 correctness 1.0의 절반 미만 → group sum sign flip 위험 낮음.
- ablation: base=0.5 (default), 1.0, 2.0 sweep. Phase 4d에서 best 결정.

### 3.6 `_build_position_teacher_batch` (round 3 R3 fix — DataProto 정확)

**v5.13 결함**: `position_ids`/`prompts`/`responses`/`response_mask` + meta_info 5 키 누락.

**v5.14 fix — `_build_teacher_logprob_batch` (verl_sdc.py:335-420) mirror, META append**:

```python
def _build_position_teacher_batch(
    *,
    tokenizer,
    prompt_texts: list[str],
    gold_answers: list[str],
    responses: torch.Tensor,        # [B, T] response tokens (response-only, prompt 분리됨)
    response_mask: torch.Tensor,    # [B, T]
    meta_start_id: int,
    rollout_temp: float,
):
    """Build teacher input = prompt + " Answer: " + gold + " " + decode(response[:p]) + META_START.

    Returns DataProto compatible with trainer._compute_ref_log_prob:
    - tensors: prompts, responses, attention_mask, response_mask, position_ids
    - non_tensor_batch: rollout_indices (mapping back to original batch)
    - meta_info: temperature, micro_batch_size, max_token_len, use_dynamic_bsz, pad_token_id
    """
    B = responses.size(0)

    # Step 1: Find first META_START position per rollout
    teacher_text_inputs: list[str] = []
    rollout_indices: list[int] = []
    for b in range(B):
        valid = (responses[b] == meta_start_id) & response_mask[b].bool()
        nz = valid.nonzero(as_tuple=True)[0]
        if nz.numel() == 0:
            continue
        p = int(nz[0].item())
        if p == 0:
            comp_text = ""
        else:
            comp_ids = responses[b, :p][response_mask[b, :p].bool()]
            comp_text = tokenizer.decode(comp_ids, skip_special_tokens=False)
        # IMPORTANT: append META_START token text so last position log_prob = log_prob(META)
        meta_token_text = tokenizer.decode([meta_start_id], skip_special_tokens=False)
        text = f"{prompt_texts[b]} Answer: {gold_answers[b]} {comp_text}{meta_token_text}"
        teacher_text_inputs.append(text)
        rollout_indices.append(b)

    if not teacher_text_inputs:
        return None  # no META emit — driver assigns zero signal

    # Step 2: Tokenize with LEFT truncation (round 3 R4.3 fix — preserve META at end)
    enc = tokenizer(
        teacher_text_inputs,
        add_special_tokens=False,
        padding="max_length",  # right-pad to max_length
        truncation=True,
        max_length=4096,
        return_tensors="pt",
    )
    # Manual left-truncation if any sequence exceeded — preserve last N tokens
    # (Note: HF tokenizer truncation_side="left" gives this directly, but we set in tokenizer init or
    #  via truncation_side parameter. v5.14: ensure tokenizer.truncation_side = "left" before this call)

    input_ids = enc["input_ids"]
    attention_mask = enc["attention_mask"]
    # position_ids: (round 3 R3 fix) — required for veRL Qwen RoPE
    position_ids = (attention_mask.cumsum(dim=1) - 1).clamp(min=0)

    # Step 3: Split into prompt + response (veRL convention).
    # For position teacher: entire sequence treated as "prompt", with response = META_START_id only.
    # last position log_prob will be log_prob(META | full input)
    N = input_ids.size(0)
    # Convention: response = last 1 token (META). prompt = preceding T-1 tokens.
    # This makes _compute_ref_log_prob return log_prob[N, 1] = log_prob(META).
    prompts = input_ids[:, :-1]  # [N, T-1]
    responses_tensor = input_ids[:, -1:]  # [N, 1] = META token
    prompt_attn = attention_mask[:, :-1]
    response_attn = attention_mask[:, -1:]
    full_attn = attention_mask  # [N, T]
    response_mask_out = response_attn.float()  # [N, 1]
    full_position_ids = position_ids

    return DataProto.from_dict(
        tensors={
            "prompts": prompts,
            "responses": responses_tensor,
            "attention_mask": full_attn,
            "response_mask": response_mask_out,
            "position_ids": full_position_ids,
            "input_ids": input_ids,  # for any path needing combined view
        },
        non_tensors={
            "rollout_indices": np.array(rollout_indices, dtype=np.int64),
        },
        meta_info={
            "temperature": rollout_temp,
            # Other 5 keys (micro_batch_size, max_token_len, use_dynamic_bsz, pad_token_id) added by trainer at dispatch
        },
    )
```

**Key trick**: response = `[META_START]` (length 1). `_compute_ref_log_prob` returns log_prob of META given full preceding context. driver extracts `ref_log_prob[:, -1]` (last response token = META) → `prob_meta`.

def prob_meta_at_position(logits, attn_mask, target_id):
    """Pure CPU function — testable verl-free."""
    last_pos = (attn_mask.sum(dim=1).long() - 1).clamp(min=0)
    B = logits.size(0)
    last_logits = logits[torch.arange(B, device=logits.device), last_pos]
    log_probs = torch.log_softmax(last_logits, dim=-1)
    return torch.exp(log_probs[:, target_id])

## 9. v5.14 self-critic + round 4 trigger

이번 patch에서 발견된 잠재 결함:

1. **§3.6 prompts/responses split**: `prompts=input_ids[:, :-1]`, `responses=input_ids[:, -1:]` 패턴이 veRL의 prompt_length 가정과 호환? `_compute_ref_log_prob` 내부에서 `prompt_length = prompts.size(1)` 가정하면 OK이나 다른 path 사용 시 문제.

2. **§3.4 prob_meta 추출 위치**: `ref_log_prob[:, -1]` — 이게 response의 마지막 token (META)에 대한 log_prob인지, prompt의 마지막 token인지? veRL convention에서 ref_log_prob shape는 `[B, response_length]`라면 -1은 response의 마지막 token = META → 정확. 그러나 shape가 다르면 (e.g., [B, full_seq]) 잘못된 위치.

3. **§3.8 tokenizer.truncation_side restore**: builder 호출 후 restore가 multi-rank dispatch에서 race condition? 각 worker가 자기 tokenizer 인스턴스 가지고 있으면 OK이나, shared면 race. veRL 0.7.1 dispatch 패턴 확인 필요.

4. **§6 RLSD invariant**: base=0.5 → max additive penalty -0.5. correctness=0 (오답) + meta_penalty=-0.2 + position_signal=-0.5 = -0.7. 그룹 평균이 +0.3 (다른 rollout 정답)이면 normalized advantage 음수 → 의도. 그룹 평균이 -0.7 이면 sign flip 위험. **Phase 4d에서 advantage sign distribution 측정 필요**.
