# SDC (Shared-Preserve Contrastive) ‚Äî Plan v3

**Date**: 2026-04-19  
**Status**: revision after `sdc-split v6` full run and eval  
**Primary variant**: `sdc-shared`  
**Comparison set**: `meta-sft`, `n3`, `sdc-split`, `sdc-uniform`, `sdc-noise`

## 1. Intent

`sdc-split v6` showed that the controller did **not** die: meta emission stayed near 1.0, but post-meta finalization degraded badly on MATH/AIME. The observed failure mode is:

1. Meta remains present.
2. Final answer closure degrades (`\boxed{}` missing or malformed).
3. AIME/MATH are hit much harder than GSM8K.
4. The likely mechanism is broad post-meta repel from `T-`, even on tokens where `T+` and `T-` agree.

So the revised intent is narrower and cleaner:

**Preserve shared post-meta structure, and apply contrast only on differential evidence.**

This targets the actual failure mode instead of assuming that broad post-meta repel is beneficial.

## 2. Hypotheses

### H1. Shared-preserve fixes finalization without killing meta
`sdc-shared` should improve post-meta finalization over `sdc-split` while keeping meta emission high.

Pass criteria:
- `no_boxed_rate` on MATH500 decreases versus `sdc-split`
- `no_boxed_rate` on AIME decreases versus `sdc-split`
- overall meta emission rate stays within 3 pp of `sdc-split`

### H2. Gains come from preserving consensus structure, not just weakening the signal
`sdc-shared` should beat `sdc-uniform` and `sdc-noise`, not merely behave like a softer regularizer.

Pass criteria:
- MATH500 accuracy > `sdc-uniform`
- AIME accuracy >= `sdc-uniform`
- MATH500 accuracy > `sdc-noise`

### H3. Shared tokens should no longer receive negative post-meta pressure
Tokens where `|log P(T+) - log P(T-)| <= tau` should be treated as teacher-consensus structure.

Verification:
- shared post-meta tokens use a consensus-preserve weight
- differential post-meta tokens use contrastive weight
- logs expose `postmeta_shared_frac` and `postmeta_diff_frac`

### H4. The method remains ‚Äúaligned epistemic shift‚Äù, not ‚Äúimitate teacher answer‚Äù
The new variant should not redefine success as copying the full teacher tail. Shared tokens are preserved because both teachers support them; differential pressure is only applied where the teachers disagree.

Failure signal:
- if shared fraction is near zero on most batches, the mechanism is not isolating consensus structure
- if finalization still collapses despite non-trivial shared fraction, the hypothesis is weakened

## 3. Verification

### Offline code-level checks
- variant dispatch includes `sdc-shared` without mutating `sdc-split`
- post-meta shared vs differential masks are disjoint and sum to post-meta
- shared tokens do not use repel-only weighting
- differential tokens do not use consensus averaging

### Smoke checks
- targeted pytest for new weighting logic
- no regression in existing SDC tests
- logs contain:
  - `meta_rlsd/postmeta_shared_frac`
  - `meta_rlsd/postmeta_diff_frac`
  - `meta_rlsd/w_shared_mean`
  - `meta_rlsd/w_diff_mean`

### Run-level checks
- compare against `sdc-split v6` on:
  - overall accuracy
  - GSM8K / MATH500 / AIME accuracy
  - meta emission rate
  - `no_boxed_rate`
  - malformed final answer rate

## 4. Failure Criteria

- `sdc-shared` leaves MATH500/AIME finalization unchanged or worse than `sdc-split`
- `sdc-shared` improves only by collapsing meta emission
- `postmeta_shared_frac` is effectively zero, meaning the mechanism is not activating
- code path silently mutates old `sdc-split` semantics

## 5. Variant Inventory

- `sdc-split`: legacy broad post-meta repel baseline
- `sdc-uniform`: magnitude-matched control
- `sdc-noise`: null directional control
- `sdc-shared`: new primary variant
  - meta region: same attract as before
  - post-meta shared tokens: preserve consensus structure
  - post-meta differential tokens: apply contrastive pressure

## 6. Execution Order

1. Keep `sdc-split v6` as the failure baseline.
2. Run `sdc-shared` first as the main repair hypothesis.
3. Only then run `sdc-uniform` / `sdc-noise` if a fresh control confirmation is needed.

## 7. Notes

This revision intentionally does **not** claim that all post-meta tokens should be repelled from `T-`. The v6 evidence argues against that claim.
