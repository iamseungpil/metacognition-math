## 0. 핵심 요약 (바쁜 독자용)

언어 모델이 **단순한 스타일 모방**이 아닌 **메타인지 컨트롤러** — 멈출지·진단할지·검색할지·답을 확정할지를 결정하는 신호 — 를 학습할 수 있는가를 묻는 연구.

- **RQ1 (Meta-CoT SFT)**: GPT-5.4-mini의 `<|meta|>` 블록 포함 trace로 학습한 Qwen3-8B는 base-matched 대조군 대비 +3.88 pp 정확도 (75.92 → 79.81 %), 메타 블록 entropy 신호 +0.300 nats로 **컨트롤러가 실제로 동작**.
- **RQ2 (Meta-RL / GDPO)**: E21R-v2 RL 300 step 후 정확도는 보존되지만 **`<|meta|>` 래핑 12 % 손실**, 신뢰도 98.9 %가 0.96에 고착, 88 %가 동일 boilerplate. 원인: `rewards.py`의 free-text fallback이 RL에 wrapping을 버려도 보상 유지하라는 신호를 줌.
- **RQ3 (자가증류)**: 메인라인 RQ3-D Epistemic Self-Distillation. **D1/D2 rebuild 완료** — controller는 복원됐지만 (Δ entropy +0.231, meta 99 %) AIME에서 47 %가 16k 토큰까지 가서 답 못 내는 **token-budget 폭주** 재발.
- **핵심 발견**: 메타 wrapping을 강제하면 → controller alive but accuracy 손실. **이 트레이드오프가 다음 두 실험 (Meta-KL meta_only / Meta-KL control_spans)** 가 풀어야 할 문제.

## 1. Research Questions

### 1.1 RQ1 — Meta-CoT는 컨트롤러인가, 스타일인가?

**의도**: `<|meta|>...<|/meta|>` 블록이 단순 장식이 아니라 **이후 reasoning을 바꾸는 게이팅 신호** 인지 검증.

**가설**: 컨트롤러가 있다면 (a) meta 포함 SFT > base-matched SFT, (b) meta 블록 직전 entropy가 높고 직후 낮음 (의도적 숙고의 신호).

**증거** (`results/strict_pair_analysis_2026_04_15`, `results/entropy_strict_meta`):
- 정확도: meta 79.81 % vs base 75.92 % (n=1030 매칭 문제)
- Entropy signature: meta span 따라 entropy +0.300 nats 변화 (n=120)
- Redirect 문제 (base가 틀린 문제): meta SFT가 +22.3 pp 더 잘 수정

**결론**: ✅ **컨트롤러 존재 확인.**

### 1.2 RQ2 — Meta-RL이 컨트롤러를 강화하는가?

**의도**: GDPO (correctness + outcome_calibration 2-head)로 RL을 돌려 컨트롤러가 더 강하게 작동하도록.

**가설**: (a) 정확도 향상, (b) wrap rate 보존, (c) redirect 품질 보존, (d) confidence 분포 다양성 보존.

**증거** (`results/step300_deep_analysis/analysis_report.md`):
- 정확도: 변화 없음 (+0.0 pp)
- Wrap rate: ~100 % → **88.2 %** (122/1030 손실)
- Confidence: **98.9 %가 0.96에 고착**
- Boilerplate: 908/1030 (88.2 %)이 동일 3-line assessment
- AIME: 평균 출력 3.2k → 11.8k 토큰, **13/30이 4096 budget 초과**
- Redirect 수정율: **−22.3 pp vs base** (역퇴화)

**결론**: ❌ **(b)(c)(d) 실패**. 메커니즘은 §4.

### 1.3 RQ3 — 컨트롤러로 test-time 정보 획득

**의도**: 발화된 컨트롤러 신호 (diagnosis + study_need)로 검색·branch 탐색·재생성을 트리거.

**서브 RQ**:
- RQ3-A: 검색 (RAG)
- RQ3-B: MCTS-lite branch 탐색
- RQ3-C: search-to-learn distill
- **RQ3-D ⭐**: Epistemic Self-Distillation (현 메인라인) — RL이 깨뜨린 컨트롤러를 reward loop 없이 재구성

**진행**: RQ3-D만 본격 진행 중. RQ3-A/B/C는 smoke 단계.

## 2. 자가증류 프레임워크

RL이 망가뜨린 컨트롤러를 self-distillation으로 복원, 또는 base 모델이 wrapping 없이도 epistemic 행동을 유지하는지 검증. **3 가지 변형** 계획.

### 2.1 변형 A — Score Teacher (Best-of-N)

**의도**: Teacher (meta SFT + 선택적 RAG)에서 N개 후보 생성, 다중 reward로 점수 매기고 best 선택. "best" = 정답 + 컨트롤러 행동 (meta wrap, diagnosis, recovery) 모두 만족.

**수식**: 문제 $q$ 에 대해 $\{c_1, ..., c_N\} \sim p_T(\cdot|q)$,
$$s(c) = \sum_k w_k \cdot r_k(c; q, a^\star)$$
6개 reward 컴포넌트의 가중합 — correctness 1.0 + confidence_revision 0.35 + redirect 0.30 + verify 0.15 + meta_floor 0.15 + meta_commit_quality 0.25. $c^\star = \arg\max_c s(c)$.

**구현**: `src/training/self_distill/online.py:245-358`

**선행 연구 비교**:
- vs **BOND** (Sessa et al. 2024) — BOND는 단일 RM, 우리는 6-component → **RLAIF-V** 계열
- vs **SPIN** (Chen et al. 2024) — SPIN은 correctness만, `selector_mode="correctness_only"`가 동일

### 2.2 변형 B — Token-Span KL (= Meta-KL)

**의도**: Teacher의 모든 토큰을 imitate하지 말고, **컨트롤러 신호 토큰만** (meta wrap, diagnosis, study_need, recovery, verify span) KL로 distill. 스타일 over-imitation 방지.

**수식**: forward-KL + top-k truncation + 위치 가중치
$$\mathcal{L}_{\text{KL}} = \frac{1}{\sum_t m_t} \sum_t m_t \cdot D_{\text{KL}}(p_T(\cdot|x_{<t}) \,\Vert\, p_S(\cdot|x_{<t}))$$
where $m_t \in \{0, 1, w_{\text{meta}}, w_{\text{diag}}, w_{\text{study}}, w_{\text{recov}}, w_{\text{verify}}\}$.

**구현**: `src/training/self_distill/kl.py:110-176`, `src/training/sft.py:280-338`, `src/training/self_distill/teacher_query.py:95-135`

**선행 연구 비교**:
- vs **MiniLLM** (Gu et al. 2024) — MiniLLM은 모든 토큰에 reverse-KL, 우리는 span 가중 forward-KL → **OPSD / GKD** 계열
- vs **GKD** (Agarwal 2023, arXiv:2306.13649) — top-k truncation + target augmentation 트릭 동일

### 2.3 변형 C — RLSD-lite (RL with self-distilled init)

**의도**: D2 SFT 체크포인트에서 시작해 verl-GDPO RL을 redirect 데이터에 돌리되, **commit-shape reward**로 no-boxed / boilerplate 반복 / decoherence 꼬리를 직접 페널티.

**구현**: `src/training/verl_reward.py:253` (`compute_score_e21r_v4_smoke` — 8개 reward 컴포넌트), `scripts/launch_e21r_v4_commit_shape_0416.sh` (lr 1e-6, KL 0.001, batch 64, 4 rollouts, 4096 토큰 응답, 300 step, TP=2)

**arXiv:2604.03128 "RLSD"와의 차이**: 정통 RLSD는 per-token teacher-ratio advantage scaling. 우리 "RLSD-lite"는 verl-GDPO + reward shaping 강화. 정통 방식은 추가 forward pass + advantage 수정 필요.

**현 상태**: ❌ **노드에 verl 환경 없음** (`/scratch/simplerl_venv` 부재). 이번 세션에서는 진행 불가, 다음 세션으로 이월.

## 3. 이전 발견 — 무엇을 알고 있는가

### 3.1 SFT 후 컨트롤러 존재
정확도 +3.88 pp, entropy Δ +0.300 nats. `data/v8_meta_inside_strict.parquet` vs `v8_base_matched_strict.parquet` 매칭쌍.

### 3.2 RL이 컨트롤러를 침식
E21R-v2 (300 step, 2-head GDPO):
- `<|meta|>` wrap rate: ~100 % → 88.2 %
- Confidence mode collapse: 0.96 가 98.9 %
- Boilerplate: 908/1030 동일 텍스트
- AIME 출력 길이 3.2k → 11.8k, 13/30 budget 초과

### 3.3 RL이 컨트롤러를 깨는 메커니즘 — `rewards.py` 의 free-text fallback
`_parse_meta_blocks_with_spans` (`rewards.py:366`)의 3단계 fallback:
1. `<|meta|>...<|/meta|>` 매칭 → 보상
2. `[META] ... [/META]` 매칭 → 보상
3. **`r"confidence\s*:\s*[\d.]+"` 매칭 → 같은 보상** ★ leakage

RL이 학습 중 **wrapping 토큰을 비용으로 인식하고 버림**. 보상은 그대로 유지됨 → wrapping 손실은 자연스러운 RL 최적화 결과.

### 3.4 Confidence collapse 메커니즘 — `outcome_calibration_reward`의 선형성
$$r_{\text{cal}}(c, \hat{y}, y^\star) = 0.3 \cdot (1 - (c - \mathbb{1}[\hat{y}=y^\star])^2) - 0.15$$
$$\mathbb{E}_{y^\star}[r_{\text{cal}}] = 0.3 \cdot c \cdot (2\,\text{acc} - 1) + \text{const}$$
acc > 0.5 일 때 $c \to 1$ 이 최적 → 정직한 $c = \text{acc}$가 아님. **proper scoring rule (Brier-log)** 면 honest 하지만, 우리 reward는 그렇지 않음.

### 3.5 RL 출력 entropy 신호
`results/entropy_analysis_step300/rl_meta_confidence/` (n=200, anchor=free-text "confidence:"):
- All: Δ −0.052 nats
- Correct: Δ −0.042
- **Incorrect: Δ −0.193** ← forced certainty (틀려도 confidence 강제 emit)

SFT와 **부호 반대** — 숙고가 아닌 강요된 확신.

### 4.8 RLSD-lite — misnomer 정정 + 이월

코드를 다시 살펴보니 **우리 `compute_score_e21r_v4_smoke`는 정통 RLSD가 아님**.

| 항목 | 정통 RLSD (arXiv:2604.03128) | 우리 "RLSD-lite" |
|---|---|---|
| Teacher signal | per-token $\log P_T(y_t \| x)$ | 없음 |
| Advantage | $A \cdot \text{clip}((P_T/P_S)^{\text{sign}(A)})$ | scalar group-relative |
| Teacher 사용 | 매 step forward pass | init weights로만 (간접) |
| 본질 | RL + teacher distribution shaping | **RL + reward shaping** |

우리 것은 그냥 GRPO + 개선된 reward shaping (`meta_commit_shape`, `structure`, `floor` 등). Token 분포 신호 없음. **이름 오용 — 실제로는 "v4-shape RL"**. verl 환경 설치 완료 (train_b 노드의 `/opt/conda/envs/ptca`)했지만 이번 세션 내 진행은 미완. 다음 세션으로 이월.

### 4.10 원인 분석 — Self-Distill이 의도대로 동작 안 하는 이유

**원인 1: Teacher = Student → 정보 이득 없음**
- 우리 teacher는 `v8_meta_inside_strict_sft` (= 학생과 동일 weights)
- Best-of-N으로 표본 추출해도 분포는 teacher capacity에 갇힘
- 정통 RLSD는 teacher가 ground truth answer로 조건화된 **privileged context** → 학생이 모르는 정보 전달
- **우리 teacher는 학생이 이미 아는 것만 가르침**

**원인 2: Data scale 차이 100배**
- D2 rebuilt: 1749 rows × 2 epoch = ~3.5k gradient steps
- RL E21R-v2: 2935 prompts × 4 rollouts × 300 step = **~3.5M rollouts**
- SFT는 train data 분포만 학습 → 작은 data + 일반화 부족

**원인 3: Exploration 없음**
- SFT: "이 sequence 따라 생성" — exploration 없음
- RL: "정답이면 reward, 다양한 출력 탐색" — exploration 있음
- AIME 같은 어려운 문제는 **exploration 없이 generalization 못 됨**

**원인 4: Best-of-N + correct filter의 비대칭**
- 어려운 문제는 8 candidates 다 틀려서 D2 rebuilt 데이터에서 통째 빠짐
- 학생은 **teacher가 풀 수 있는 쉬운 문제만 학습** → AIME 같은 hard에선 일반화 못 됨
- D1 (correctness only, meta strip)이 AIME +6.7pp 성과 낸 이유: meta 강제 없이 다양한 데이터 학습

**원인 5: Meta wrap 강제 = reasoning 토큰 budget 잡아먹음**
- D2 rebuilt: 모든 응답 meta wrap → AIME 47% 16k 초과 (truncation)
- Meta wrapping = 100-200 토큰 overhead × 여러 번 = reasoning 길이 직접 감소

### 4.11 개선 방향 (다음 세션 권장)

**방향 1: Privileged Teacher** ⭐ (가장 정통 RLSD 정신)
- Teacher prompt에 ground truth answer 주입: "정답은 X. 메타 wrap reasoning 만들어라"
- 학생이 모르는 정보 (어떻게 정답에 도달하나) 전달
- 코드 변경: `teacher_query.py`의 system/user prompt 수정만

**방향 2: 정통 RLSD Token-level Scaling**
$$\hat{A}_t = A \cdot \text{clip}\left(\left(\frac{P_T(y_t|x_{<t})}{P_S(y_t|x_{<t})}\right)^{\text{sign}(A)}, 0.8, 1.2\right)$$
- 매 토큰별 teacher advantage scaling
- 코드 작업 1-2일 (`grpo_v2.py`에 teacher hook)

**방향 3: Hybrid SFT → RL**
- D3b 체크포인트 (controller alive, acc 낮음) → 추가 RL
- RL이 정확도 회복 + small KL to ref가 controller 보존
- 이미 코드 있음 (`compute_score_e21r_v4_smoke` + verl 설치됨)

**방향 4: Iterative Self-Improvement**
- Round 1 SFT → 더 좋은 teacher → Round 2 SFT → ...
- Self-Refine / Constitutional AI 스타일

**방향 5: Data scale + Diversity 폭증**
- 외부 teacher (GPT-5, Claude) 추가
- N=8 → N=32
- RL rollouts 자체를 correctness filter 후 self-distill data로

**방향 6: Curriculum + Wrapping 분리**
- Stage 1: base SFT on hard data (D1 rebuilt 패턴) — 정확도 ↑
- Stage 2: 별도로 meta wrapping만 학습
- 두 신호 분리로 trade-off 회피

**방향 7: Reasoning length budget을 reward에 직접 추가**
- 현재 `meta_commit_shape`는 "no boxed"만 페널티
- 추가: "16k 도달 시 큰 페널티", "AIME reasoning 12k 초과 시 감점"

**우선순위**: 방향 1 (코드 1일) > 방향 3 (코드 있음, KL_COEF 조정) > 방향 2 (큰 효과 but 큰 작업).

## 5. 코드 충실도 감사

§4의 발견을 토대로 **5개 이슈 우선순위**:

### 🔴 5.1 Reward free-text fallback (`rewards.py:366`)
`allow_free_text_fallback=True` 기본값. 10+ 호출처가 free-text confidence를 메타 블록과 동일 보상으로 처리 → wrapping loss 직접 원인.
**조치**: `False` 기본값으로 변경 + per-call 재검토.

### 🟡 5.2 Easy 제외 필터 과도 (`trace.py:249`)
`is_easy = difficulty in ("easy", "trivial", "")` — 빈 문자열도 easy 처리 → 라벨 없는 root-correct 모두 탈락. AIME/MATH 일부에 영향.
**조치**: `("easy", "trivial")` strict 비교 + 라벨 없을 때 warn.

### 🟡 5.3 Length 편향 selector (`online.py:336`)
`(is_correct, score, -len(completion))` 정렬 — 동점에서 긴 답 선호 → boilerplate 강화.
**조치**: 길이 tiebreaker opt-in.

### 🟡 5.4 Token-KL alignment drift (`kl.py:110-176`, `sft.py:280-338`)
Teacher token vs label token 불일치 시 silent skip. 채트 템플릿 차이로 모든 위치 drop 가능.
**조치**: 첫 mismatch 시 warn + chat template equivalence 감사.

### 🟡 5.5 [META] plain-text 불일치
`kl.py:19-22`는 `<|meta|>` 만 인식, rewards는 `[META]` 까지 fallback → KL과 reward 커버리지 불일치.
**조치**: §5.1 fix가 동시에 해소.

### ⚪ 5.6 RLSD 미구현 (정통 방식)
arXiv:2604.03128 per-token teacher-ratio scaling은 우리 코드에 없음. 우리 RLSD-lite는 verl + reward shaping 변형.
