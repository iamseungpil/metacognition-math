# RQ3-D2b Dry-Run Autoresearch (2026-04-15)

## Goal

Verify that `RQ3-D2b` is no longer only provenance logging and now has a
code-backed, falsifiable path for `offline feedback-conditioned OOD recovery distill`.

## Scope

1. `src/training/self_distill_data.py`
2. `scripts/build_self_distill_dataset.py`
3. `scripts/analyze_self_distill_eval.py`
4. `scripts/analyze_rq3_side_eval.py`
5. `src/curriculum/rq3_pipeline.py`
6. `tests/test_self_distill_data.py`
7. `tests/test_rq3_pipeline.py`
8. `scripts/smoke_rq3_self_distill_path.py`

## Metric

The dry-run succeeds only if all of the following hold:

1. `sdpo_regen` build path exists and writes non-empty parquet on a valid RQ3 repaired trace
2. privileged feedback is injected into train-time prompt context, not only side columns
3. `analyze_rq3_side_eval.py` can compute the D2b gate metrics:
   - OOD combined accuracy delta
   - trigger precision drop
   - next-meta recovery rate
   - optional `teacher_feedback_available_rate`
4. test and smoke guards pass

## Guard

1. `python -m py_compile` on touched files
2. `pytest -q tests/test_self_distill_data.py tests/test_rq3_pipeline.py tests/test_control_rag.py`
3. `python scripts/smoke_rq3_self_distill_path.py`

## Current Status

What is now supported:

1. `D1`: naive collapse probe
2. `D2a`: offline epistemic-preserving controller retention
3. `D2b`: offline feedback-conditioned recovery distill with explicit train-time conditioning
4. falsifiable D2b gate readout for side-eval summaries

What is still not supported:

1. SDPO-style token-wise teacher-conditioned distillation
2. any claim that D2b already equals dense self-distillation
3. any claim of OOD success before a real `D1 vs D2b` training/eval pair is run
