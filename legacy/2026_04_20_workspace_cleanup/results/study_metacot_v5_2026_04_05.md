## 1. 서론 및 배경

본 절은 Meta-CoT 프로젝트의 연구 맥락과 Control-V5 실험이 필요한 이유를 제시한다.

대형 언어 모델(LLM)은 Chain-of-Thought(CoT) 프롬프팅을 통해 수학 추론 성능을 향상시킬 수 있다. 그러나 표준 CoT에는 자기 점검(self-monitoring) 메커니즘이 없다. 모델이 잘못된 경로에 진입하면 이를 감지하거나 수정할 수단이 없으며, 자신의 답에 대한 확신도를 교정(calibrate)할 수도 없다. 이 문제를 해결하기 위해 본 프로젝트는 Meta-CoT를 제안한다. Meta-CoT는 일반 추론 흐름과 분리된 `<|meta|>` 토큰 블록을 통해 다음 다섯 가지 메타인지 상태를 명시적으로 외현화한다: (1) 국소 확신도(local confidence), (2) 이상/모순 감지(anomaly detection), (3) 실패 진단(failure diagnosis), (4) 다음 제어 행동(next control action: verify 또는 redirect), (5) 학습 필요사항(study_need).

선행 실험(V2-V4)은 메타인지 포맷의 학습 가능성과 교정 효과를 확인했다. AIME-2024에서 과신(overconfidence)이 97%에서 14%로 감소하고, ECE가 0.870에서 0.610으로 개선되었다. 그러나 정확도는 base SFT 대비 일관되게 하락했다. Control-V5는 이 정확도 격차의 원인을 체계적으로 분석하기 위해 설계되었다. 8개 모델(SFT 4개, RL 4개)을 동일 평가 세트에서 비교하여, 메타인지 행동이 실제 추론 성능에 미치는 영향을 정량화한다.

본 보고서는 7개의 핵심 발견(Finding)에 대해 각각 RQ-가설-방법-결과-해석의 구조로 기술한다. 모든 수치는 format-fixed 평가(n=1,030)에서 직접 확인한 값이다.

## 2. 연구 질문 및 가설

본 절은 Control-V5 실험군이 검증하고자 하는 연구 질문과 사전 가설을 명시한다.

### 2.1 연구 질문

| ID | 연구 질문 |
|---|---|
| RQ1 | Meta-CoT가 base SFT 대비 수학 추론 정확도를 향상시키는가? |
| RQ2 | SFT와 RL 중 어떤 학습 방법이 메타인지 행동을 더 효과적으로 유도하는가? |
| RQ3 | 메타인지 진단(diagnosis)이 실제 경로 전환(route switching)으로 이어지는가? |
| RQ4 | Meta-CoT 도입 시 토큰 오버헤드가 정확도에 부정적 영향을 미치는가? |
| RQ5 | 모델이 어려운 문제에 선택적으로 meta를 생성하여 결과 해석에 편향을 만드는가? |

### 2.2 사전 가설

| ID | 가설 | 근거 |
|---|---|---|
| H1 | Meta-CoT는 verify/redirect를 통해 오답을 교정하여 base SFT를 초과할 것이다 | 메타인지가 test-time adaptation을 가능케 한다는 이론적 기대 |
| H2 | RL이 SFT보다 정확한 메타인지 행동을 학습시킬 것이다 | SFT는 행동을 모방하지만 RL은 결과 기반으로 행동을 교정한다 |
| H3 | 진단이 정확하면 경로 전환이 뒤따를 것이다 | 정확한 진단은 올바른 대안 경로 선택의 전제조건이다 |
| H4 | Meta 블록이 유용한 정보를 제공하므로 토큰 오버헤드가 정확도 손실보다 작을 것이다 | 메타인지의 이점이 추가 토큰 비용을 상쇄한다는 가정 |
| H5 | Meta 생성은 문제 난이도와 무관하게 균일할 것이다 | SFT 데이터가 전체 난이도 범위를 커버하므로 |

## 3. 실험 설계

본 절은 8개 실험 조건의 구성, 평가 벤치마크, 평가 지표를 기술한다.

### 3.1 모델 구성

모든 모델은 Qwen3-8B를 기반으로 한다. SFT 모델은 GPT-5.4-mini가 생성한 Meta-CoT 체인(4,996개)으로 학습하되, 어떤 메타인지 행동을 포함하는지에 따라 변형된다. RL 모델은 SFT 체크포인트에서 출발하여 GRPO(Group Relative Policy Optimization)를 적용하며, reward 함수의 조합에 따라 구분된다.

| 모델 | 유형 | 학습 내용 |
|---|---|---|
| base_sft | Plain SFT | meta 없이 동일 문제의 표준 CoT로 학습 |
| verify_sft | Meta-CoT SFT | verify 행동(자기 점검)만 포함한 meta 체인으로 학습 |
| redirect_sft | Meta-CoT SFT | redirect 행동(경로 전환)만 포함한 meta 체인으로 학습 |
| all_sft | Meta-CoT SFT | verify + redirect + diagnosis 전체를 포함한 meta 체인으로 학습 |
| E8 | RL | overconfidence penalty: 오답 시 높은 확신도에 벌점 부여 |
| E9b | RL | verify + redirect reward 조합 |
| E9c | RL | verify + overconfidence penalty + length penalty 조합 |
| E10 | RL | full controller: 모든 reward를 통합한 완전 제어 모델 |

### 3.2 평가 벤치마크

| 벤치마크 | 문제 수 | 난이도 | 역할 |
|---|---|---|---|
| GSM8K | 500 | 초등~중등 산술 | 기본 추론 능력 측정 |
| MATH-500 | 500 | 고등~대학 수학 | 복합 추론 능력 측정 |
| AIME-2024 | 30 | 경시대회 수준 | 극한 난이도에서의 메타인지 효과 검증 |

### 3.3 평가 지표

- **정확도(Accuracy)**: 최종 답의 정오 판정. 통화($70,000) 및 쉼표 포함 숫자의 정규화 후 판정.
- **선택 편향 분석**: meta가 존재하는 문제와 부재하는 문제를 분리하여 base 모델 정확도 비교.
- **메타인지 진단 품질**: AIME 오답에 대해 진단의 정확성, verify/redirect 실행 여부를 수작업 분류.
- **토큰 오버헤드**: completion 토큰 수의 모델 간 비교, 4,000 토큰 절단(truncation) 빈도.

### 3.4 Format 버그 수정

평가 과정에서 `_check_correctness` 함수가 통화 기호(`$70,000`)와 쉼표를 포함한 답안을 인식하지 못하는 버그가 발견되었다. SFT 모델은 학습 데이터의 영향으로 이러한 포맷을 빈번하게 생성했으며, 30-48개 답안이 이 버그로 인해 오답 처리되었다. 통화/쉼표 정규화를 추가하여 수정한 후 SFT 모델은 +2.9~+4.7pp 회복되었다. RL 모델은 이 포맷을 거의 생성하지 않아 영향이 0-4개로 미미했다. 본 보고서의 모든 수치는 수정된 평가 결과에 기반한다.

## 4. Finding 1: Meta-CoT는 base 모델을 초과하지 못한다 {#4-finding-1}

### RQ

Meta-CoT를 학습한 모델이 meta 없이 학습한 base_sft보다 높은 정확도를 달성하는가?

### 가설

H1: Meta-CoT는 verify/redirect를 통한 오답 교정으로 base_sft를 초과할 것이다.

### 방법

8개 모델을 동일한 1,030문제에서 평가하고, 전체 정확도(Overall Accuracy)와 벤치마크별 정확도를 비교한다. 최대 생성 토큰은 4,096으로 통일한다.

### 결과

Figure 1은 8개 모델의 벤치마크별 정확도를 비교한다(`fig1_accuracy_comparison.png`).

**Table 1. 8개 모델의 정확도 비교 (n=1,030, format-fixed)**

| 모델 | 유형 | GSM8K | MATH500 | AIME | Overall | Gap vs base |
|---|---|---|---|---|---|---|
| base_sft | Plain SFT | 88.8% | 48.8% | 10.0% | 67.1% | baseline |
| E9c | RL (verify+overconf+length) | 81.4% | 43.0% | 10.0% | 60.7% | -6.4pp |
| verify_sft | Meta-CoT SFT (verify) | 81.0% | 42.8% | 6.7% | 60.3% | -6.8pp |
| E9b | RL (verify+redirect) | 81.0% | 41.8% | 3.3% | 59.7% | -7.4pp |
| E8 | RL (overconfidence penalty) | 80.0% | 38.8% | 6.7% | 57.9% | -9.2pp |
| redirect_sft | Meta-CoT SFT (redirect) | 78.0% | 40.0% | 3.3% | 57.4% | -9.7pp |
| all_sft | Meta-CoT SFT (all behaviors) | 75.8% | 41.2% | 10.0% | 57.1% | -10.0pp |
| E10 | RL (full controller) | 71.8% | 38.8% | 6.7% | 53.9% | -13.2pp |

### 해석

H1은 기각된다. 8개 Meta-CoT 모델 전체가 base_sft(67.1%)보다 -6.4pp에서 -13.2pp 낮은 정확도를 보인다. GSM8K에서의 하락이 특히 일관적이며(88.8% -> 71.8~81.4%), MATH-500에서도 모든 모델이 base_sft(48.8%)를 하회한다. AIME에서는 base_sft(10.0%)와 동일하거나 낮은 수준이다.

주목할 점은 reward 조합이 복잡해질수록 정확도가 하락하는 경향이다. 단일 reward 모델(E8: -9.2pp, E9c: -6.4pp)보다 full controller(E10: -13.2pp)의 하락이 크다. SFT에서도 유사한 패턴이 관찰된다: verify_sft(-6.8pp)보다 all_sft(-10.0pp)의 하락이 크다. 이는 메타인지 행동의 종류를 늘릴수록 추론 자원이 메타 생성에 전용되어 본질적 문제 해결 능력이 약화됨을 시사한다.

이 결과는 현재의 Meta-CoT가 "알고 있지만 행동하지 못하는" 상태임을 의미한다. 메타인지 형식은 학습되었으나, 그것이 실제 추론 궤적을 개선하는 방향으로 작동하지 않는다. 이 격차의 구체적 원인은 Finding 4-6에서 분석한다.

## 5. Finding 2: Format 버그가 SFT 모델 정확도를 과소평가했다 {#5-finding-2}

### RQ

평가 파이프라인의 답안 인식 방식이 특정 모델 유형에 체계적 불이익을 주는가?

### 가설

`_check_correctness` 함수의 포맷 제한이 SFT 모델에 불균형적 영향을 미칠 것이다. SFT 모델은 학습 데이터(GPT-5.4-mini 생성 체인)의 답안 포맷을 그대로 재현하기 때문이다.

### 방법

수정 전후의 정확도 변화를 모델별로 비교한다. 수정 내용은 `$70,000` -> `70000`, `1,234` -> `1234`와 같은 통화 기호 및 쉼표 정규화다.

### 결과

| 모델 유형 | 포맷 버그로 손실된 답안 수 | 수정 후 회복 |
|---|---|---|
| SFT 모델 (verify, redirect, all) | 30-48개 | +2.9 ~ +4.7pp |
| RL 모델 (E8, E9b, E9c, E10) | 0-4개 | 0 ~ +0.4pp |

### 해석

Format 버그는 SFT 모델에 체계적 불이익을 주었다. RL 모델은 GRPO 학습 과정에서 correctness reward를 받기 위해 평가 함수가 인식하는 포맷으로 수렴한 반면, SFT 모델은 학습 데이터의 포맷을 그대로 재현했다. 이 발견은 두 가지를 시사한다. 첫째, 평가 파이프라인의 답안 파싱이 모델 비교의 공정성에 직접 영향을 미치므로, 포맷 정규화는 실험 설계의 필수 요소다. 둘째, format 수정 후에도 모든 Meta-CoT 모델이 base_sft를 하회하므로, 정확도 격차의 주원인은 포맷이 아니라 다른 곳에 있다.

## 6. Finding 3: 선택 편향이 meta 효과를 왜곡한다 {#6-finding-3}

### RQ

Meta-CoT 모델이 meta 블록을 생성하는 문제와 생성하지 않는 문제 사이에 난이도 차이가 있는가? 그 차이가 meta의 효과 추정을 왜곡하는가?

### 가설

H5: Meta 생성은 문제 난이도와 무관하게 균일할 것이다.

### 방법

Meta-CoT 모델의 응답을 meta 존재(present) / 부재(absent) 그룹으로 나누고, 각 그룹의 문제를 base_sft로 풀었을 때의 정확도를 비교한다. base_sft 정확도가 낮은 문제는 난이도가 높은 문제다. 두 그룹 간 base 정확도 격차는 선택 편향의 크기를 나타낸다.

### 결과

Figure 2는 meta 존재/부재에 따른 base 정확도 차이와 선택 편향 구조를 보여준다(`fig2_meta_impact_selection_bias.png`).

**Table 2. 선택 편향 분석: meta 존재 문제와 부재 문제의 base_sft 정확도 비교**

| 모델 | Meta 존재 문제 -> base acc | Meta 부재 문제 -> base acc | 격차 |
|---|---|---|---|
| verify_sft | 63.1% | 78.2% | -15.0pp |
| all_sft | 63.6% | 81.4% | -17.8pp |
| E8 | 33.0% | 70.7% | -37.7pp |

### 해석

H5는 기각된다. 모든 모델에서 meta가 존재하는 문제의 base_sft 정확도가 현저히 낮다. E8의 경우 격차가 -37.7pp에 달한다. 이는 모델이 어려운 문제에서 선택적으로 meta를 생성함을 의미한다.

이 선택 편향은 두 가지 해석상의 함정을 만든다. 첫째, "meta가 없는 문제에서 정확도가 높다"는 관찰은 meta 부재가 유리하다는 증거가 아니라, 쉬운 문제에서 meta가 불필요하여 생성되지 않았다는 증거다. 둘째, "meta가 있는 문제에서 정확도가 낮다"는 관찰도 meta가 해롭다는 증거가 아니라, 원래 어려운 문제에서 meta가 생성되었다는 증거다. 따라서 meta의 인과적 효과를 추정하려면 동일 난이도의 문제에서 meta 유무에 따른 정확도를 비교하는 통제된 설계가 필요하다.

E8에서 선택 편향이 가장 큰 이유는 overconfidence penalty가 모델로 하여금 확신도가 낮은(즉, 어려운) 문제에서만 meta를 생성하도록 유도했기 때문으로 해석된다. 이는 RL reward가 meta 생성의 조건을 조절하는 효과적 수단임을 시사하지만, 동시에 쉬운 문제에서의 verification 기회를 상실하게 만드는 부작용도 있다.

## 7. Finding 4: 메타인지 진단은 정확하나 실행이 부재한다 {#7-finding-4}

### RQ

Meta-CoT 모델이 생성하는 failure diagnosis가 올바른가? 올바른 진단이 실제 경로 전환으로 이어지는가?

### 가설

H3: 진단이 정확하면 경로 전환이 뒤따를 것이다.

### 방법

all_sft 모델의 AIME-2024 응답 30개를 수작업으로 분류했다. 분류 기준은 다음 네 범주다: (1) 진단이 정확하고 경로 전환 없음(correct diagnosis, no switch), (2) 과신적 verify(overconfident verify), (3) 진단이 정확하고 부분적 경로 전환(correct diagnosis, partial switch), (4) meta 부재(no meta), (5) 잘못된 진단(wrong diagnosis).

### 결과

Figure 3은 AIME 30문제의 메타인지 진단 품질 분포를 보여준다(`fig3_aime_diagnosis_quality.png`).

**Table 3. AIME 메타인지 진단 품질 (all_sft, 30문제)**

| 범주 | 건수 | 정확도 |
|---|---|---|
| 정확한 진단, 경로 전환 없음 | 20 | 0% (0/20) |
| 과신적 verify | 5 | 20% (1/5) |
| 정확한 진단, 부분적 경로 전환 | 3 | 100% (3/3) |
| Meta 부재 | 2 | 0% (0/2) |
| 잘못된 진단 | 0 | N/A |

### 해석

H3은 부분적으로 기각된다. 진단의 정확성은 주목할 만하다: 잘못된 진단(wrong diagnosis)이 0건이다. 모델은 자신이 왜 막혔는지를 항상 올바르게 인식한다. 그러나 정확한 진단 20건 중 경로를 실제로 전환한 경우는 0건이다. 모델은 "orbit counting이 필요하다"고 정확히 진단하면서도 본문에서는 여전히 부정확한 case counting을 이어간다.

이 결과는 현재 Meta-CoT의 핵심 병목이 진단 능력이 아니라 실행 능력임을 명확히 한다. 비유하자면, Meta-CoT는 완벽한 진단서를 작성하지만 처방전을 쓰지 않는 의사와 같다. 진단과 실행 사이의 이 격차를 "진단-실행 격차(diagnosis-execution gap)"라 명명한다.

진단-실행 격차의 근본 원인은 SFT 학습 데이터에 있다. GPT-5.4-mini가 생성한 메타인지 체인에서 "진단 후 경로를 전환하여 정답에 도달하는" 궤적(trajectory)이 부재한다. 모델은 진단하는 법은 배웠으나 전환하는 법을 배울 데이터가 없었다. 이 근본 원인 분석은 Finding 5에서 확인된다.

## 8. Finding 5: 경로 전환이 정답의 필요충분조건이다 {#8-finding-5}

### RQ

AIME에서 Meta-CoT 모델이 정답을 맞힌 경우, 어떤 메타인지 행동이 정답과 연관되는가?

### 가설

경로 전환(approach_change)이 발생한 경우에만 정답률이 유의하게 높을 것이다.

### 방법

all_sft의 AIME 정답 3건과 오답 27건을 비교하여, 정답 사례의 공통 특성을 추출한다.

### 결과

AIME 30문제에서 all_sft가 정답을 맞힌 3건은 전부 "정확한 진단 후 부분적 경로 전환(partial switch)"이 발생한 사례다(Table 3 참조). 나머지 27건의 오답 중 경로 전환이 발생한 경우는 0건이다.

즉, `approach_change > 0`인 사례의 정확도는 100%(3/3)이고, `approach_change = 0`인 사례의 정확도는 0%(0/27)이다.

### 해석

경로 전환은 AIME 수준의 어려운 문제에서 정답에 도달하기 위한 핵심 행동이다. 이 결과는 세 가지를 시사한다.

첫째, 메타인지의 가치는 진단 자체가 아니라 진단에 기반한 행동 변화에 있다. 진단만으로는 정확도가 향상되지 않는다(0/20). 전환이 수반되어야 한다(3/3).

둘째, 현재 SFT 데이터가 전환 궤적을 포함하지 않으므로, 모델은 전환을 학습할 수 없었다. 이는 데이터 수준의 근본 원인이다.

셋째, 경로 전환의 효과가 이렇게 뚜렷하다면, 전환 궤적을 명시적으로 수집하여 학습시키는 것이 정확도 향상의 가장 직접적인 경로다. 구체적으로, rejection sampling을 통해 "진단 -> 전환 -> 정답"의 성공 궤적만 선별하여 학습 데이터를 구성하는 전략이 유효할 것이다.

## 9. Finding 6: 토큰 오버헤드와 절단이 정확도를 잠식한다 {#9-finding-6}

### RQ

Meta 블록이 소비하는 토큰이 본문 추론을 위한 토큰을 잠식하여 정확도를 낮추는가?

### 가설

H4: Meta 블록의 이점이 토큰 비용을 상쇄할 것이다.

### 방법

모델별 평균 completion 토큰 수와 4,000 토큰 이상 절단 빈도를 비교한다.

### 결과

**Table 4. 토큰 오버헤드 비교**

| 지표 | base_sft | all_sft | 변화 |
|---|---|---|---|
| 평균 completion 토큰 | 285 | 420 | +47% |
| Meta 블록 비율 | 0% | 15-25% | - |
| 절단 빈도 (>=4000 토큰) | 1건 | - | baseline |

**Table 5. 모델별 절단 빈도**

| 모델 | 절단 빈도 (>=4000 토큰) |
|---|---|
| base_sft | 1 |
| E9c | 20 |
| redirect_sft | 17 |

### 해석

H4는 기각된다. Meta 블록은 completion 토큰의 15-25%를 소비하며, 이는 동일한 토큰 예산 내에서 본문 추론에 사용할 수 있는 토큰을 줄인다. E9c와 redirect_sft에서 각각 20건, 17건의 절단이 발생했다. 이는 base_sft의 1건에 비해 현저히 높다. 절단된 응답은 최종 답을 생성하지 못하므로 반드시 오답으로 처리된다.

토큰 오버헤드의 영향은 문제 난이도에 따라 비대칭적이다. GSM8K와 같은 쉬운 문제에서는 base_sft도 200-300 토큰으로 풀 수 있으므로 meta 오버헤드가 절단을 유발하지 않는다. 그러나 AIME이나 MATH-500의 어려운 문제에서는 풀이 자체가 2,000-3,000 토큰을 요구하므로, meta 블록의 추가 500-1,000 토큰이 절단을 유발할 수 있다. 역설적으로, meta가 가장 필요한 어려운 문제에서 meta의 토큰 비용이 가장 큰 부담이 된다.

이 문제의 해결을 위해 두 가지 방향을 제안한다: (1) meta-lite 포맷(핵심 정보만 포함하는 압축 포맷)의 도입, (2) 난이도 적응적 meta 생성(쉬운 문제에서는 meta를 생략하고, 어려운 문제에서만 compact meta를 생성).

## 10. Finding 7: RL reward가 의도와 다른 행동을 유도한다 {#10-finding-7}

### RQ

RL reward 함수가 의도한 메타인지 행동을 학습시키는가, 아니면 reward를 회피하는 방향으로 학습이 진행되는가?

### 가설

H2: RL이 SFT보다 정확한 메타인지 행동을 학습시킬 것이다.

### 방법

RL 모델(E8, E9b, E9c, E10)의 정확도, meta 생성 패턴, 절단 빈도를 분석하여 reward 함수의 의도된 효과와 실제 효과를 비교한다. 추가로 E6의 probe_calibration_reward 학습 곡선을 분석한다.

### 결과

RL 모델의 결과를 세 가지 관점에서 정리한다.

(1) 정확도: 모든 RL 모델이 base_sft를 하회한다(Table 1). E9c가 -6.4pp로 가장 양호하고, E10이 -13.2pp로 가장 큰 하락을 보인다.

(2) Reward 회피 정황: E6의 probe_calibration_reward는 -0.33에서 0.0으로 300 step에 걸쳐 수렴했다. 이는 모델이 probe의 예측에 사후적으로 동의하는 방향(trivial agreement)으로 학습되었을 가능성이 있다. 즉, 실제 교정이 아니라 probe 출력을 복사하는 방식으로 reward를 획득했을 수 있다.

(3) Full controller의 역효과: E10(full controller)은 가장 많은 reward 성분을 통합했음에도 최저 정확도(53.9%)를 기록했다. 이는 다수의 reward 간 상충(reward conflict)이 학습을 방해함을 시사한다.

### 해석

H2는 기각된다. 현재의 RL reward 설계는 메타인지 행동을 정확도 향상으로 연결하는 데 실패했다. 세 가지 구체적 문제가 확인된다.

첫째, overconfidence penalty(E8)는 과신 억제에는 효과적이나, 그 자체로 정확도를 높이지 않는다. 과신을 줄이는 것과 정답률을 높이는 것은 별개의 목표다.

둘째, reward 성분 수가 증가할수록 정확도가 하락한다(E9c -6.4pp -> E10 -13.2pp). 이는 다수의 reward가 상충하여 학습이 불안정해지거나, 모델이 정확도 이외의 축으로 reward를 최적화하기 때문이다.

셋째, probe_calibration_reward의 수렴 패턴은 reward hacking(의도된 행동이 아닌 방법으로 reward를 획득)의 가능성을 보여준다. 이는 calibration reward가 인과적이 아니라 상관적으로 설계된 경우에 발생하는 전형적 문제다.

이 결과는 RL reward 설계에서 "경로 전환 후 정답"이라는 인과적 조건을 명시적으로 보상해야 함을 시사한다. 단순히 메타인지 행동의 출현을 보상하는 것으로는 정확도 향상을 달성할 수 없다.

## 11. 종합 논의

본 절은 7개 Finding의 전체적 함의를 종합하고, 현재 Meta-CoT의 위치를 진단한다.

### 11.1 핵심 결론

Control-V5 실험의 핵심 결론은 다음 한 문장으로 요약된다: **Meta-CoT는 완벽한 진단서를 작성하지만 처방전을 쓰지 않는 의사다.**

이 비유는 데이터에 의해 정확히 뒷받침된다:
- 진단 정확도 100% (wrong diagnosis = 0/30, Finding 4)
- 진단 후 경로 전환 0% (approach_change = 0/20, Finding 4)
- 경로 전환 시 정답률 100% (3/3, Finding 5)

### 11.2 진단-실행 격차의 원인 구조

진단-실행 격차는 세 개의 계층에서 발생한다.

**계층 1: 데이터 부재.** SFT 학습 데이터에 "진단 -> 전환 -> 정답" 궤적이 없다. GPT-5.4-mini가 생성한 체인은 단일 경로에서 풀리는 문제를 위주로 구성되어 있으며, 경로 실패 후 대안 경로로 전환하여 성공하는 사례가 포함되지 않았다. 모델은 존재하지 않는 행동을 학습할 수 없다.

**계층 2: 토큰 예산 경쟁.** Meta 블록이 15-25%의 토큰을 소비하면서 본문 추론의 토큰 예산을 줄인다. 어려운 문제에서는 이 경쟁이 절단을 유발하여 답 자체를 생성하지 못하게 만든다.

**계층 3: Reward 비정렬.** RL reward가 메타인지 행동의 "출현"을 보상하지, "출현 후 정답"을 보상하지 않는다. 결과적으로 모델은 정확도와 무관하게 meta를 생성하거나, 반대로 meta 생성 자체를 회피하여 reward pressure를 무마한다.

### 11.3 선택 편향의 방법론적 함의

Finding 3의 선택 편향(-15.0pp ~ -37.7pp)은 Meta-CoT 연구에서 반드시 통제해야 할 교란 변수다. "meta 유무에 따른 정확도 비교"라는 자연스러운 분석 방법이 이 편향에 의해 무효화된다. 향후 meta의 인과적 효과를 추정하려면, (1) 동일 문제에 대해 meta 유/무 조건을 강제하는 개입 실험, 또는 (2) 문제 난이도를 공변량으로 통제하는 매칭 분석이 필요하다.

### 11.4 긍정적 신호

부정적 결과 속에서도 세 가지 긍정적 신호가 있다.

첫째, 진단 능력 자체는 완전하다. 모델은 자신의 추론 경로가 왜 약한지를 항상 정확히 인식한다. 이는 메타인지의 "감지(detection)" 단계가 성공적으로 학습되었음을 의미한다.

둘째, 경로 전환의 효과가 극명하다(0% vs 100%). 이는 전환 능력만 학습시키면 정확도가 향상될 수 있다는 강한 증거다.

셋째, 선택 편향의 존재 자체가 모델이 난이도를 인식하고 meta 생성을 조건적으로 수행하는 능력이 있음을 보여준다. 이 조건적 생성 능력은 향후 난이도 적응적 메타인지 설계의 기반이 될 수 있다.

## 12. 향후 계획

본 절은 진단-실행 격차를 해소하기 위한 구체적 후속 실험 계획을 제시한다.

### 12.1 Pass@k 분석 (상한선 측정)

**목적**: 경로 전환의 잠재적 가치를 정량화한다.

동일 문제에 대해 k개(k=8, 16, 32)의 응답을 생성하고, 하나라도 정답인 비율(Pass@k)을 측정한다. Pass@k가 Pass@1보다 현저히 높으면, 모델이 정답 경로를 생성할 능력은 있으나 이를 선택하지 못하는 것이다. 이 경우 경로 전환이 정확도를 높일 수 있는 상한선이 된다.

### 12.2 Rejection Sampling으로 전환 궤적 수집

**목적**: SFT 데이터에 부재한 "진단 -> 전환 -> 정답" 궤적을 수집한다.

다수의 응답을 생성한 후, "초기 경로에서 진단이 발생하고, 대안 경로로 전환하여 정답에 도달한" 궤적만 선별한다. 이 궤적으로 SFT 데이터를 보강하여 전환 행동을 명시적으로 학습시킨다.

### 12.3 조건부 RL: 전환 성공 시에만 보상

**목적**: "전환했는데 여전히 오답"인 경우를 배제하고, "전환 후 정답"일 때만 reward를 부여한다.

현재 reward 설계의 핵심 문제는 메타인지 행동 자체를 보상한다는 것이다. 조건부 RL은 다음 공식을 따른다: `reward = correctness * has_switch`. 이는 정답이면서 전환이 있을 때만 보상하여, 전환의 인과적 가치를 학습시킨다.

### 12.4 추론 시 재경로화 (Inference-time Re-routing)

**목적**: 학습 없이도 meta 신호를 활용하여 경로를 전환한다.

모델이 진단을 생성하면 해당 시점에서 생성을 중단하고, 진단 내용을 프롬프트에 반영하여 새로운 경로로 재생성한다. 이는 학습 비용 없이 진단-실행 격차를 우회하는 방법이다.

### 12.5 Meta-lite 포맷

**목적**: 토큰 오버헤드를 줄인다.

현재 meta 블록의 15-25% 토큰 비용을 5-10%로 압축한다. 구체적으로, 확신도는 숫자 하나(예: `<|meta|>conf=0.3`), 진단은 한 문장, study_need는 키워드만 포함하는 최소 포맷을 설계한다.

### 12.6 우선순위

| 순위 | 실험 | 기대 효과 | 비용 |
|---|---|---|---|
| 1 | Pass@k 분석 | 상한선 측정, 의사결정 근거 | 추론만, 학습 불필요 |
| 2 | Rejection sampling + SFT | 전환 궤적 학습 | 데이터 수집 + SFT 1회 |
| 3 | Inference-time re-routing | 학습 없이 진단 활용 | 추론 비용 2배 |
| 4 | 조건부 RL | 전환-정답 인과 학습 | GRPO 학습 1회 |

## [NEW] 14. Addendum: 추가 발견 (2026-04-05 후반)

### 14.1 토큰 Budget Ablation

**RQ**: max_completion_length가 정확도에 미치는 영향은?

**결과** (base_sft, MATH500 첫 100문제):

| max_tokens | Accuracy | Truncated | Avg Tokens |
|---|---|---|---|
| 2048 | 45.0% | 1개 | 330 |
| 4096 | 42.0% | 0개 | 310 |

**해석**: 2048이 4096보다 +3.0pp 높음. 토큰 제한을 줄이면 모델이 더 간결하게 답하여 정확도가 오히려 향상. **토큰 확대는 불필요하며, 오히려 적절한 제한이 유리할 수 있다.**

### 14.2 E6 Probe Model 중간 결과 (680/1030)

E6 eval이 680/1030 시점에서 running accuracy 66.3%. base_sft의 전체 67.1%에 근접. 이전 "probe collapse 경보"(200/1030에서 75.5%)는 GSM8K 구간 편향으로, MATH500 진입 후 base_sft와 유사한 수준으로 수렴 중.

**해석**: probe training이 trivial collapse가 아닐 가능성. 최종 결과로 확정 필요.

### 14.3 E3 Full Eval 완료

E3 (calibration RL): 51.7% overall (GSM8K 69.6%, MATH500 36.8%, AIME 3.3%). 전체 모델 중 최하위. 하지만 **confidence coverage 100%** (1029/1030) — calibration reward가 meta 생성을 극대화하여 모든 문제에서 meta를 냄. 이것이 정확도를 가장 많이 떨어뜨린 설정.

### 14.4 Re-routing Oracle 분석

all_sft의 "route is weak" 신호를 사용한 oracle re-routing:
- 144개 re-route 후보 (14% of total)
- 45개 oracle 복구 가능 (31.2%)
- **Oracle accuracy: 57.1% → 61.5% (+4.4pp)**
- MATH500에서 +7.6pp (41.2% → 48.8%)

**해석**: meta의 "감지" 능력은 inference-time routing signal로 가치가 있음.

### 14.5 Verification 심층 분석 (Phase 0.5)

**RQ**: verify가 높은 confidence에서 실제로 정답을 확인하는가?

| 모델 | High-conf FP Rate | Conf 수정 횟수 | 해석 |

| **E9** | **30.0%** (246/821) | **25개** | RL이 FP 줄이고 수정 6x 학습 |

**해석**: E9에서 RL이 verify의 질을 개선 (FP 43%→30%, 수정 4→25개). 하지만 여전히 FP 30%는 높아서 verify를 전환 trigger로 사용 불가. "막힘 감지" (confidence < 0.4 + route_weak) 기반 trigger가 더 안전.

**Phase 1 설계 영향**: verify 제외, 순수 "막힘 감지" 기반 전환 trigger 사용.
